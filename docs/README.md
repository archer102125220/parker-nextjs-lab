## 📂 文檔使用指南

### 新加入開發者
1. 先閱讀 [completed/implementation_plan.md](./completed/implementation_plan.md) 了解專案整體架構
2. 查看 [in-progress/task.md](./in-progress/task.md) 了解當前進度和優先事項
3. 參考 [completed/bem-naming-guide.md](./completed/bem-naming-guide.md) 了解 CSS 命名規範
4. 根據分配的任務查閱對應的詳細文檔

### 日常開發
1. 查看 [in-progress/task.md](./in-progress/task.md) 確認當前任務
2. 參考 [completed/next-steps-guide.md](./completed/next-steps-guide.md) 規劃每日工作
3. 完成任務後更新對應的任務清單

### CSS 重構工作（階段 2 - 當前進行中）
1. 查看 [in-progress/phase-2-progress.md](./in-progress/phase-2-progress.md) 了解當前進度
2. 使用 [in-progress/phase-2-task-tracking.md](./in-progress/phase-2-task-tracking.md) 追蹤任務
3. 參考 [completed/bem-naming-guide.md](./completed/bem-naming-guide.md) 了解命名規範
4. 完成每個頁面後更新檢查清單
5. 查看 [completed/phase-2-verification-report.md](./completed/phase-2-verification-report.md) 了解驗證標準

### CSS 重構工作（階段 1 - 待執行）
1. 閱讀 [pending/css-compliance-fix-plan.md](./pending/css-compliance-fix-plan.md) 了解修復計劃
2. 使用 [in-progress/css-compliance-fix-task.md](./in-progress/css-compliance-fix-task.md) 追蹤進度
3. 完成每個任務後更新檢查清單

### API 開發和測試
1. 參考 [in-progress/api-testing-results.md](./in-progress/api-testing-results.md) 了解測試狀態
2. 執行測試並記錄結果
3. 發現問題時更新文檔

## 🔄 文檔維護

### 更新頻率
- **in-progress/task.md**: 每日更新
- **in-progress/phase-2-progress.md**: 每完成一個頁面更新
- **in-progress/phase-2-task-tracking.md**: 每完成一個頁面更新
- **in-progress/css-compliance-fix-task.md**: 每完成一個任務更新
- **in-progress/api-testing-results.md**: 每次測試後更新
- **completed/next-steps-guide.md**: 每週檢視並更新
- **completed/implementation_plan.md**: 重大變更時更新
- **README.md**: 本文件，新增/移除文檔時更新

### 文檔分類規則
- **completed/**: 已完成且不需頻繁更新的文檔
- **in-progress/**: 正在進行中，需要持續更新的文檔
- **pending/**: 計劃中但尚未開始的文檔

### 文檔命名規範
- 使用小寫字母和連字符
- 使用描述性名稱
- 添加適當的後綴（-plan, -task, -guide, -results, -report）
